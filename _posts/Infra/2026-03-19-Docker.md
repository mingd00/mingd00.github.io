### Docker?

A technology that **packages applications and infrastructure** together to ensure **consistent execution in any environment**.

- **application**: code with logic
- **infrastructure**: the environment for execution of the application

The application always depends on the infrastructure.

Errors can occur even when running the same code if the server or runtime version is different, and a lot of resources are wasted to align these infrastructure environments.

Docker solves this problem.

Docker packages these two into a single container and **eliminates the difference between the development and execution envirnoments**.

### Docker Features

- **immutable infrastructure**: the state of a server at a specific point in time can be saved and replicated
- **portability**: consistent results are produced across different environments
- **idempotency**: the result does not change even if the same operation is repeated multiple times
- **easy management**
- **fast deployment speed**

### The deployment process of a Docker container

"Create a Dockerfile -> Build an Image -> Run a Container"

Docker manages applications and infrastructure together in a single container.
An image is a template for creating a container, and the build process of the image is defined in a Dockerfile.
